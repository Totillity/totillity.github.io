Timeline for Project
Week 1: Initial Design

  1. Look up previous work on JIT’s and how they work

  2. Based on that work, create a plan for how my JIT will work

Week 2: Prototype Assembler

  1. Read up on x86 Assembly (the one used in my computer)

  2. Create a basic Assembler based on the reading

Week 3: IR Design

  1. Read up on IR designs

  2. Look into what my IR will need

  3. Design an IR to fulfill those needs

Week 4: Implement IR

  1. Implement the previous IR into my assembler

  2. Test that IR

  3. Ensure I fully implement the IR

Week 5: Design Frontend

  1. Look through current languages and see what I want mine to look like

  2. Based on my previous work, I can see what language design would lend itself best to JITing

  3. Design such a language

Week 6: Implement Frontend

  1. Implement Parsing for the language

  2. Research better error handling and ast representation. Advisor could help a lot here.

  3. Learn how to do the above in C

Week 7: Hook up Frontend and Backend, General improvements #1

  1. Implement a way to generate the IR from the frontend

  2. Out of all the steps so far, this is the most likely to either fail or work poorly.

  3. Therefore, also use this week to implement all the improvements and fixes I’ve thought of over the past 7 weeks

Week 8: Optimization Techniques, Part 1

  1. Look up some optimization techniques, first non-JIT specific

  2. Whenever I find one I want to implement, do so

Week 9: Optimization Techniques, Part 2

  1. Look up some optimization techniques, this week preferably JIT specific

  2. Whenever I find one I want to implement, also do so

Week 10: Python Transpiler

  1. Find a way to transpile Python to my JIT

  2. Possibly use the ‘ast’ python module

Week 11: Profile Guided Optimization

  1. Run programs on my JIT, see where the biggest slowdowns are

  2. Slow downs in the JIT itself can be optimized

  3. Slow downs from the JIT not optimizing can also be corrected by researching and implementing more optimizations

Week 12: General Improvements #2

  1. Implement the other ideas that came to me during the past 12 weeks

  2. For example, I think I’ll add support for Windows here, which will likely just mean adding another calling convention

Week 13: Professional-ize project

  1. Clean-up code

  2. Write tests

  3. In my site, write documentation on how the JIT works, and how to use it

Week 14: Final Week

  1. This week might be a result of my miscount. In which case, shift this workload a week earlier

  2. Otherwise, prep for my final project presentation.

  3. Also, a flex week for the work that might be postponed due to Tests and other sources of stress
